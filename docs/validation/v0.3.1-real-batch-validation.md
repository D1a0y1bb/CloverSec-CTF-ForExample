## 本轮暴露并修复的问题

- picoCTF Forensics 查询里，CSDN 的 Web/NoSQL 文章分数偏高。已增加分类冲突识别，`web`、`nosql`、`sql`、`injection`、`xss`、`ssti`、`csrf`、`ssrf` 会作为 Web 信号；查询 Forensics 时这些结果会带 `category mismatch: web` 并降权。
- `source-evidence` 遇到 HTTP 403 时原本 `errors=0`，容易误导。已改为 HTTP 4xx/5xx 计入 `errors`，并在对应 evidence 的 `missing_reason` 写入 `http_status=403`。
- 真实 LA CTF 样本中 `challenge.yaml`、`solve.sage`、`flag.txt` 起初被标成 unknown。已增加 CTF manifest、Sage 解题脚本和 flag 文件识别。
