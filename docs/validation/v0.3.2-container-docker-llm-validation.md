Issue found and fixed during validation:

- Docker build plan used `-f Dockerfile` while running from repo cwd, so Docker could not find the temporary Dockerfile.
- `container_inference.runtime.container_command` incorrectly reused Dockerfile `CMD` and caused `docker run` to override the image command.
- Fixed by resolving Dockerfile paths against `project_dir` and keeping Dockerfile `CMD` as metadata instead of a `docker run` override.
