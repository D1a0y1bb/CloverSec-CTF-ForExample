## Issues Found And Fixed

1. `cloversec_ctf_manual_quality.py` failed when optional `--archive-manifest` and `--resource-manifest` were not provided.
   - Error: `TypeError: argument should be a str or an os.PathLike object ... not 'NoneType'`.
   - Fix: `load_optional_json` now accepts `None`.
   - Added CLI regression test.

2. `cloversec_ctf_hub.py draft` failed when `archive.screenshots[]` used the normal dict shape `{"path": "..."}`.
   - Error: `TypeError` from `Path(item)` where `item` was a dict.
   - Fix: Hub packaging now accepts string paths and dict paths for attachments and screenshots.
   - Added regression test.

3. `hub_draft.json` did not include the `summary` field described by the data model.
   - Fix: `hub_draft.json` now includes `summary.status`, `summary.can_fill_browser`, `summary.can_submit`, `summary.validation_status`, blockers, pending uploads, and `manual_path`.

4. The first real LLM test showed that a model without the exact ID list may omit IDs or invent `mcp__...` names.
   - Fix: plugin `defaultPrompt` now keeps short boundary rules, and exact ID routing is kept in `agents/openai.yaml` plus `references/agent-roles.json`.

5. New `codex exec` session exposed a Codex manifest warning: `interface.defaultPrompt` supports at most 3 prompts and each prompt must be at most 128 characters.
   - Fix: `plugin.json` now uses exactly 3 short prompts.
   - `scripts/validate_release.py` now checks this limit before release.
