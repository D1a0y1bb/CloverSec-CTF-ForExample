# 输出规范

本文件定义 `CloverSec-CTF-Writeup-Scaffold` 的固定输出结构、枚举值和格式约束。

## 1. 固定章节

输出必须包含以下章节：

- `1 题目设计部署信息`
- `1.1 题目名称`
- `1.2 题目描述`
- `1.3 题目难度`
- `1.4 考察信息`
- `1.5 旗帜信息`
- `1.6 附件情况`
- `1.7 部署方式`
- `1.7.1 Dockerfile 构建启动`
- `1.7.2 Tar 镜像包导入启动`
- `1.8 非预期测试`
- `2 HUB上传部分&题解信息`
- `2.1` ~ `2.14`

## 2. 固定枚举

### 题目难度

只允许：

- `简单`
- `中等`
- `困难`

### 题目分类

只允许：

- `Web`
- `Pwn`
- `Crypto`
- `Misc`
- `Reverse`
- `Mobile`
- `IoT`

### Flag 类型

只允许：

- `静态Flag`
- `动态Flag`

### 题目来源

只允许：

- `原创`
- `搬运`

### 题目类型

只允许：

- `环境型`
- `附件型`

## 3. 格式约束

### HUB 录题字段

`2.1` ~ `2.11` 必须使用 `plain` 代码块。

### 分隔符

`2.11` 与 `2.12` 之间必须插入 `---`。

### 附件列表

- 顶层使用 `●`
- 子项使用 `○`
- 路径、命令、英文术语使用行内代码

### 部署方式

`1.7.1` 与 `1.7.2` 需要提供命令示例，放在代码块中。

### Flag 信息

`1.5` 默认包含：

- 原始静态 flag 信息
- `flag` 文件位于容器根目录 ` /flag ` 的说明
- 平台支持动态 flag 分发与写入约束的说明

## 4. 推荐写法

- 用中文固定层级结构输出
- 描述偏工程事实，不做营销文案化扩写
- 正式手册优先写成人工整理后的交付稿，不输出模板感说明。
- 证据不足的解题步骤要写明待人工确认，不能编造。

## 5. 当前版本默认输出

- `分类-题目名.md`
- `题目解题手册.md`
