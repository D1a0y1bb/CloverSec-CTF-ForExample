# 推断边界说明

本文件定义哪些信息可以保守推断，哪些信息必须留空或使用占位符。

## 可以推断的内容

当存在明确证据时，可以保守推断：

- 分类：依据 `challenge.yaml` 的栈、题目目录命名、WP 标题前缀、源码类型
- 题目类型：存在 `Dockerfile`、`start.sh` 等环境交付件时，通常判为 `环境型`
- 资源等级：若题目来源明确为 `原创`，可填 `8`；明确为 `搬运`，可填 `4`
- 部署命令：可根据 `Dockerfile`、`start.sh`、端口与镜像名称生成示例命令
- 附件结构：可根据当前目录的顶层目录和关键文件生成层级清单
- 关键词：可基于“考察知识点”“前置知识”“栈信息”提炼少量关键词

## 不可凭空补写的内容

没有明确证据时，以下内容必须留空或放占位符：

- 题目名称
- 题目描述
- 静态 flag
- 题目来源
- 题目分值
- 题目等级 / 星级展示内容
- 非预期测试点
- 题目备注中的平台细节
- 详细解题步骤

## 推荐证据来源顺序

1. `challenge.yaml`
2. `Dockerfile` / `start.sh` / `changeflag.sh` / `check/`
3. 用户显式提供的 WP / Markdown / 说明文档
4. 当前工作目录下为完成任务必需的源码与配置文件

## Flag 推断原则

- 如果文本中明确出现 `flag{...}`、`ctf{...}` 等静态 flag 形态，可填入静态 flag
- 如果存在 `changeflag.sh`、动态 flag 写入说明或平台动态 flag 契约，可把 `Flag类型` 倾向于 `动态Flag`
- 如果两类证据都不完整，优先留空或使用占位符，不强行判断

## 语气原则

- 不写“根据分析可以看出这是一个非常典型的……”这类 AI 套话
- 优先短句、事实句、整理稿口吻
- 推断结论尽量落到具体文件、端口、命令、路径或知识点上
